기본 규칙

프로젝트 목적

  • 기존 PC버전 e-Accounting 시스템의 화면/기능을 Mobile 환경에 맞게 확장

모바일 기준

  • 해상도 375 기준 / 360 사이즈는 틀어지지 않는 정도
  • 태블릿 사이즈 750~1280 에서는 max-width:750px center로 고정

HTML 문서 작성 기본 규칙

  • DTD : HTML5
  • 인코딩 : utf-8

Naming 규칙

  • CSS
    Class명은 BEM 방법론을 따른다 (http://getbem.com/naming/)
    기본정의 : block요소name--Modifier(변형자 or 상태)__Element(자식요소. key와 value는 하이픈(-)사용)
    .부모명__자식 , .부모명__자식--확장 , .부모명__자식명구분-자식명구분, .부모명__자식명 .확장타입—타입속성
    Ex).product__list, product__list—disabled, .product__list-item, .product__list.text-align--left
  • image
    이미지타입_메뉴관련네이밍_확장숫자.확장자
    ex)ico_product_item_01.png

css 작업 방법

  • scss 작업 후 vscode의 llive sass compiler 설치 후 Watch Sass 실행하여 compile 된 css를 html에 연결
  • resource > scss 내의 파일이 빌드되어 css > common.css 파일로 최종 빌드됨
  • ※ 개발단에 css 공유 시 scss compile 후 css 파일 가장 하단의 /*# sourceMappingURL=.....map */ 삭제하여 전달해야 개발단에서 여러번 불러오지 않음
  • 운영시에는 scss가 아닌 css 파일을 직접 수정하는 방향으로 산출물에 scss 제외하여 전달
