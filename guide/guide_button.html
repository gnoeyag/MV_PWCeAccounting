본문 바로가기

case 1 : 플로팅 되지 않고 콘텐츠 하단에 존재하는 버튼

  • ※ jQuery로 콘텐츠 감싸는 영역 높이, 콘텐츠 내용 높이 비교하여 .non-fixed class 추가/제거 하는 방식으로 default position:fixed 상태에서 스크롤 될 경우 position:relative 상태로 변경
  • ※ 개발 시 검색/탭 sorting 등 액션이 실행 될 때, 값을 뿌려준 뒤 버튼영역 위치 조절을 위하여 js 코드 호출 필요
  • 해당 버튼 사용 할 경우 바닥페이지의 경우.content, 레이어팝업의 경우.layer-content 영역에 .padding-bottom--btn-wrap class 추가
  • 최초 진입 시 콘텐츠 내용이 콘텐츠 감싸는 영역 높이 보다 작을 경우 디바이스 하단에 버튼 위치 (하단 position:fixed 상태)
  • 최초 진입 시 콘텐츠 내용이 콘텐츠 감싸는 영역 높이 보다 클 경우 콘텐츠 스크롤 후 하단에 버튼 위치 (position:relative 상태로 변경)
  • 조회,검색 등 무언가의 액션으로 콘텐츠 내용의 높이에 변화가 생길 경우 위의 내용과 같이 버튼 위치 변동

case 2 : 플로팅 버튼 layout_fixed.html에서 안내

  • .floating-btn 영역에 더블 class로 버튼1개,2개 일 경우 여백값 구분
  • 버튼 1개로 상하좌우 여백 있는 경우 : default 추가 class 없음
  • 버튼 2개로 상하좌우 여백 없는 경우 : .radius-non-type

  • 최초 진입 시, 페이지 내 클릭 등의 액션을 통하여 나온 결과값이 .content 영역의 height 값이 디바이스높이보다 클 경우 : .btn-wrap.btn-wrap--bottom 영역에 .non-fixed class 추가
  • 최초 진입 시, 페이지 내 클릭 등의 액션을 통하여 나온 결과값이 .layer-content 영역의 height 값이 디바이스높이보다 클 경우 : .btn-wrap.btn-wrap--bottom 영역에 .non-fixed class 추가
  • 반대로 최초 진입 시 아무것도 동작하지 않았으나 페이지 내 클릭 등의 액션을 통하여 나온 결과값이 디바이스 높이보다 작을 경우 추가 한 .non-fixed class 삭제
